# 要件定義書：BackHub（バックハブ）

## 1. プロジェクト概要

- **サービス名**: BackHub
- **目的**: 複数プロジェクトにまたがるBacklogの課題を1画面で横断的に一覧・管理でき、チームおよび全社の業務効率を向上させる社内向けWebサービス。
- **背景**: Backlogの標準機能では複数プロジェクトの課題一覧性が低く、スプレッドシートでの手動管理もスケールしないため、システム化によって解決する。

## 2. ターゲットユーザー

- **初期**: 開発チームメンバー（エンジニア中心）
- **最終**: 全社展開（プロジェクトマネージャー、ディレクターなど全職種）

## 3. システムアーキテクチャ・技術スタック

- **フロントエンド**: React, Next.js
- **スタイリング/UI**: Tailwind CSS, shadcn/ui
- **バックエンド (BFF)**: Node.js (初期データ取得・バッチ処理用)
- **データベース**: Supabase (PostgreSQL, リアルタイム同期機能を利用)
- **インフラ / ホスティング**: Vercel (フロントエンド及びサーバーレス関数), Vercel Cron Jobs

## 4. 機能要件 (MVP)

### 4.1. 認証・認可機能

- **SSOログイン**: BacklogのOAuth2を利用したシングルサインオン。

### 4.2. データ同期・取得機能

- **初回データフェッチ**: ユーザーの初回ログイン後、プロジェクト設定完了時に、Node.jsのBFFを介して対象プロジェクトの「直近3ヶ月分」の課題をBacklog APIから取得しDB(Supabase)に保存。フロントエンドは取得完了までローディング（またはスケルトン）表示を行う。
- **リアルタイム同期**: BacklogのWebhookを受信し、SupabaseのDBを更新。Supabaseのリアルタイム通信を利用して、フロントエンドの画面をリロードなしで即時更新する。

### 4.3. 課題一覧表示機能 (メイン画面)

- **レイアウト**: 1カラム構成。長文の課題タイトルでも視認性を損なわない広さを確保。
- **表示項目**: プロジェクト名（グループヘッダーとして表示）、課題ID、課題タイトル、担当者、状態、優先度、備考。
- **リンク機能**: 一覧の課題行をクリックすると、別タブでBacklogの該当課題ページへ遷移する。
- **インライン編集機能 (備考欄)**:
  - 備考欄のテキスト横にペンアイコンを配置。
  - クリックでテキスト入力フォームに切り替わり、Enterキー押下で内容を保存（DB更新）。

### 4.4. 検索・絞り込み・並び替え機能

- **フリーワード検索**: ヘッダー上部の検索窓から、課題の「タイトル」および「課題ID」で絞り込みが可能。
- **グローバルフィルター**:
  - プロジェクト切り替え（全プロジェクト / 個別プロジェクト）
  - ステータス絞り込み（Open, In Progress, Closed などのトグル・チェックボックス）
- **プロジェクト固有フィルター (オーバーレイサイドバー)**:
  - プロジェクト名の横にある歯車アイコンをクリックで、右側からサイドバーがオーバーレイ表示される。
  - そのプロジェクト固有の「担当者」「課題種別」「マイルストーン」をチェックボックスで選択可能。
  - 「Update Filters（更新）」ボタン押下で一覧表に反映される。
- **並び替え (ソート)**:
  - プロジェクトヘッダー付近のソートアイコンから操作。
  - デフォルトは「更新日が新しい順」。

## 5. 非機能要件

- **パフォーマンス**: 初回フェッチ時以外は、APIの無駄打ちを避け、Supabaseのリアルタイム機能でサクサク動くSPAの恩恵を最大限活かす。
- **レスポンシブ対応**: 今回のMVPではPCブラウザ（フルHD等）での利用を主眼とするが、将来的なモバイル対応も考慮したコンポーネント設計（shadcn/ui）を行う。
