# BackHub 実装フェーズ計画

## フェーズ1: フロントエンドの足場固めとUIモック作成

**目的**: アプリケーションの骨組みと見た目を完成させ、UI/UXの操作感を検証する。

- Next.js, Tailwind CSS, shadcn/ui のプロジェクトセットアップ。
- 1カラム構成のメインレイアウト作成。
- 課題一覧のデータグリッド（表）をハードコード（モックデータ）で作成。
- ヘッダー（検索窓、ユーザーアイコンなど）の作成。
- 右側からのオーバーレイサイドバー（歯車アイコン押下時のフィルター）のガワを作成。

## フェーズ2: データベース設計とSupabaseセットアップ

**目的**: データを永続化するためのバックエンド基盤を構築する。

- Supabaseプロジェクトの作成。
- データベーススキーマの設計とテーブル作成（Users, Projects, Issues, Settingsなど）。
- RLS（Row Level Security）の基本設定（今回は社内ツール想定なので緩めでも可）。
- フロントエンドからSupabaseのモックデータを取得・保存できるかテスト接続。

## フェーズ3: 認証機能の組み込み (Google / GitHub OAuth)

**目的**: ユーザーが安全にログインし、個人やプロジェクトに紐づくデータを扱えるようにする。

- Google OAuth2 および GitHub OAuth2 のアプリケーション登録。
- NextAuth.js (Auth.js) または Supabase Auth を用いたGoogle / GitHubログインの実装。
- ログイン状態に応じた画面の出し分け（未ログイン時はログイン画面へリダイレクト等）。
- ログイン後のBacklog APIキー登録画面の実装（ユーザーごとにAPIキーを保存）。

## フェーズ4: バックエンドBFF構築と初期データ取得

**目的**: 実際のBacklogデータをアプリケーションに取り込み、UIを本物のデータで描画する。

- Node.js (またはNext.jsのAPI Routes) でBFF（Backend For Frontend）を構築。
- Backlog APIを叩き、対象プロジェクトの「直近3ヶ月分」の課題を取得するロジックの実装。
- 取得したデータをSupabaseのIssuesテーブルに保存（Upsert処理）。
- フェーズ1でハードコードしたUIを、Supabaseから取得した本物のデータに置き換える。
- データの取得待ち時間中のローディング（スケルトンスクリーン）UIの実装。

## フェーズ5: リアルタイム同期の実現

**目的**: Backlog側の更新を画面のリロードなしで即座に反映させる。

- BacklogのWebhook設定（課題の追加・更新・削除時にBFFへ通知を飛ばす）。
- Webhookを受け取り、Supabaseのデータを更新するAPIエンドポイントの実装。
- フロントエンドでSupabaseのRealtime subscriptionsを設定し、DBの変更を検知してUIを自動更新する処理の実装。

## フェーズ6: 検索・絞り込み・ソート機能の実装

**目的**: 大量に取得した課題データを、ユーザーが目的の条件で探せるようにする。

- クライアントサイド（またはSupabaseクエリ）でのフリーワード検索機能（タイトル・ID）の実装。
- グローバルフィルター（ステータスのトグル、プロジェクト切り替え）の状態管理と連動。
- プロジェクト固有フィルター（サイドバーでの担当者、種別、マイルストーンのチェックボックス）の実装と「Update Filters」ボタンによる反映ロジック。
- カラムヘッダー（更新日など）をクリックした際の並び替え（ソート）機能の実装。

## フェーズ7: 備考欄のインライン編集機能

**目的**: ユーザーがBackHub上で直接情報を追記できるUXを実現する。

- 備考欄のペンアイコン押下でテキストボックスに切り替わるUIコンポーネントの作成。
- Enterキー押下時（またはフォーカスが外れた時）に入力内容をSupabaseに保存（UPDATE）する処理。
- 保存時のローディング表示と、エラー時のトースト通知（shadcn/uiのToast等）の実装。

## フェーズ8: インフラ構築・デプロイと最終調整

**目的**: 本番環境にデプロイし、実際の運用に耐えうる状態にする。

- Vercelへのデプロイ設定（環境変数の登録など）。
- 必要に応じたVercel Cron Jobsの設定（Webhookの取りこぼしを防ぐための定期同期バッチなど）。
- パフォーマンスチューニング（不要な再レンダリングの防止など）。
- 最終的な全体テストとバグフィックス。
